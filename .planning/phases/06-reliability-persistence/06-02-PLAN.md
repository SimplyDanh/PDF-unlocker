# Plan: Chunked Worker API & Stream Implementation (06-02)

## 🛠️ Tasks

<task id="1">
<name>Implement Chunked Read in pdfWorker.js</name>
<files>
- `services/pdfWorker.js`
- `tests/pdfWorker.test.js`
</files>
<action>
1. Define `CHUNK_SIZE = 64 * 1024 * 1024` (64MB).
2. Define `STREAM_THRESHOLD = 250 * 1024 * 1024` (250MB).
3. In `processFile`, after QPDF execution, check output file size via `FS.stat`.
4. If size > `STREAM_THRESHOLD`, use `FS.open`, `FS.read` in a loop, and `FS.close`.
5. For each chunk, `postMessage` with `type: 'chunk'`, `chunkIndex`, `totalChunks`, and the `Uint8Array.buffer` as a Transferable Object.
6. If size <= `STREAM_THRESHOLD`, keep the existing `FS.readFile` logic for performance.
</action>
<verify>
<automated>
npm test tests/pdfWorker.test.js
</automated>
</verify>
<done>
The worker correctly switches to chunked streaming for large files and sends chunks as Transferable Objects.
</done>
</task>

<task id="2">
<name>Update Persistence Service for Chunk Storage</name>
<files>
- `services/persistenceService.js`
</files>
<action>
1. Update IndexedDB schema (v2) to include a `chunks` store.
2. `chunks` store: `id` (auto-increment), `fileId` (index), `index`, `data` (Blob/Uint8Array).
3. Implement `saveChunk(fileId, index, data)` and `getChunks(fileId)`.
4. Implement `assembleFileFromChunks(fileId)` which returns a single `Blob`.
</action>

<task id="3">
<name>Integrate Chunked Receiver in WorkerPool</name>
<files>
- `services/pdfService.js`
</files>
<action>
1. Update `setupWorker` in `WorkerPool` to handle the `chunk` message type.
2. When a chunk is received, call `persistenceService.saveChunk`.
3. Update `handleWorkerSuccess` to detect if the result was streamed (no blob in success message).
4. If streamed, use `persistenceService.assembleFileFromChunks` to provide the final Blob to the UI.
</action>
