# 05-02: Integrity & Policy Hardening

## Tasks

<task>
<read_first>
- "index.html"
- "ui/app.js"
</read_first>
<action>
Create `scripts/generate-sri.js`. It should use `crypto` to generate SHA-384 base64 hashes for:
- `ui/app.js`
- `ui/styles.css`
- `services/pdfService.js`
- `services/auditService.js`
- `services/pdfWorker.js`
</action>
<acceptance_criteria>
- `scripts/generate-sri.js` produces a list of file paths and their `sha384-...` strings.
- Hashes match manual `shasum` calculation for the same files.
</acceptance_criteria>
</task>

<task>
<read_first>
- "index.html"
</read_first>
<action>
Update `index.html`. Add `integrity` and `crossorigin="anonymous"` to all `<script src="...">` and `<link rel="stylesheet">` tags. Update the CSP `<meta>` tag to:
`default-src 'self'; script-src 'self' 'wasm-unsafe-eval'; style-src 'self' https://fonts.googleapis.com; font-src https://fonts.gstatic.com; connect-src 'self'; img-src 'self' data:; object-src 'none';`
</action>
<acceptance_criteria>
- All local asset tags have `integrity` attributes.
- App loads successfully without CSP violations in the console.
- Manual alteration of `ui/app.js` (e.g., adding a space) prevents the script from loading due to integrity failure.
</acceptance_criteria>
</task>

<task>
<read_first>
- "tests/security.spec.js"
</read_first>
<action>
Create `tests/security.spec.js`. Implement Playwright tests that:
1. Verify that a script with an invalid SRI hash is blocked.
2. Verify that `eval()` is blocked by CSP.
3. Verify that requests to unauthorized external domains (e.g., `https://evil.com`) are blocked.
</action>
<acceptance_criteria>
- `npx playwright test tests/security.spec.js --grep "@csp"` passes.
- Test report shows clear failure/success for each policy requirement.
</acceptance_criteria>
</task>
