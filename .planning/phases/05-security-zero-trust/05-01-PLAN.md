# 05-01: Advanced Security Logic & Persistence

## Tasks

<task>
<read_first>
- "services/pdfService.js"
</read_first>
<action>
Create `services/auditService.js`. Implement a singleton `AuditService` that uses IndexedDB to store records in a `logs` store within `PDF_UNLOCKER_DB` (v1). Define `logEvent(type, details)` where `type` is one of 'INFO', 'SUCCESS', 'ERROR' and `details` is a JSON object.
</action>
<acceptance_criteria>
- `services/auditService.js` exists.
- `indexedDB.open('PDF_UNLOCKER_DB')` succeeds in console.
- `AuditService.logEvent('INFO', { test: true })` adds a record to the `logs` store.
</acceptance_criteria>
</task>

<task>
<read_first>
- "services/pdfWorker.js"
</read_first>
<action>
Update `services/pdfWorker.js` to use `self.crypto.subtle.digest('SHA-256', outputBuffer)` on the decrypted PDF buffer. Return the hex-encoded string as a `hash` property in the message sent back to the main thread.
</action>
<acceptance_criteria>
- `pdfWorker.js` contains `crypto.subtle.digest`.
- Worker response message includes a 64-character hex string in the `hash` field.
</acceptance_criteria>
</task>

<task>
<read_first>
- "services/pdfService.js"
- "services/auditService.js"
</read_first>
<action>
Import `AuditService` in `services/pdfService.js`. Update the file processing handler to call `AuditService.logEvent` with the file name, status, and the `hash` received from the worker. Ensure catch blocks log an 'ERROR' event with the error message.
</action>
<acceptance_criteria>
- `pdfService.js` calls `AuditService.logEvent` inside the worker message handler.
- Audit log contains an entry for every successful decryption with a valid SHA-256 hash.
</acceptance_criteria>
</task>
